# Claude Code Guidelines

## ルール

### 絵文字の使用禁止

このリポジトリでは絵文字を一切使用しないでください。

コード、コメント、ドキュメント、コミットメッセージ、メール本文、件名など、すべての箇所で絵文字の使用は禁止されています。

代わりに以下のような記法を使用してください:

- 強調: `*テキスト*` または `【テキスト】`
- 箇条書き: `*` または `-`
- 区切り線: `=` または `-` の繰り返し

### 言語とコメント

- すべての出力、説明、ドキュメントは日本語で記述してください
- コード内のコメントも日本語で記述してください
- 変数名や関数名は英語で構いませんが、説明的なコメントは日本語を使用してください
- ユーザーへの応答も日本語で行ってください

### コミットメッセージのルール

コミット時は以下のプレフィックスを使用してください:

- `fix`: バグ修正
- `hotfix`: クリティカルなバグ修正
- `add`: 新規（ファイル）機能追加
- `update`: 機能修正（バグではない）
- `change`: 仕様変更
- `clean`: 整理（リファクタリング等）
- `disable`: 無効化（コメントアウト等）
- `remove`: 削除（ファイル）
- `upgrade`: バージョンアップ
- `revert`: 変更取り消し

例: `fix: ダークテーマのボタン色を修正`

### コミットとプッシュ

一定の変更を行った際は、必ずコミットとプッシュを実行してください。

以下のコマンドを使用します:

```bash
# Gitパスを追加（PowerShellの場合）
$env:Path += ";C:\Program Files\Git\bin"

# ディレクトリに移動
cd "c:\Users\Tangeken\Documents\GitHub\property_info"

# 変更をステージング
git add .

# コミット（適切なプレフィックスとメッセージを使用）
git commit -m "fix: バグの説明"

# リモートにプッシュ
git push
```

コミット対象となる変更:
- 新規ファイルの作成
- 既存ファイルの編集
- ファイルの削除
- 機能の追加や修正
- バグ修正
- スタイルやUIの変更
- ドキュメントの更新
